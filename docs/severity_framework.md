# Severity Framework

## Philosophy

Severity depends on context, confidence, impact, and deployment environment.

## Severity Levels

1. Informational
2. Low
3. Low-Medium
4. Medium
5. Medium-High
6. High
7. High-Critical
8. Critical

## Context Matters

The same failure may have different severity in:

- Personal assistant use
- Business workflows
- Safety-critical systems
- Cybersecurity tooling

## Example

A factual hallucination in a casual chat may be Medium.

The same hallucination in a business or security workflow may be High or Critical.

## Future Scoring Inputs

- Failure type
- Confidence level
- Potential impact
- Repeatability
- User context
- Deployment context
